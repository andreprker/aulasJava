Palavra-chave extends em Java: Herança de Classes

Descubra a palavra-chave extends em Java, fundamental para implementar a herança e promover a reutilização de código entre classes.

O que é a Palavra-chave extends?

Em Java, a palavra-chave extends é usada para indicar que uma classe está **hereditando** de outra classe. Ela estabelece uma relação de herança entre uma **subclasse** (ou classe filha) e uma **superclasse** (ou classe pai).

Quando uma classe `B` `extends` uma classe `A`, significa que `B` adquire todos os atributos e métodos acessíveis de `A`, podendo também adicionar seus próprios atributos e métodos, ou sobrescrever (modificar) os métodos herdados. Isso é conhecido como a relação "é um tipo de" (is-a relationship).

Como extends Implementa a Herança?

A herança em Java é um dos pilares da Programação Orientada a Objetos (POO) e é implementada diretamente pelo uso da palavra-chave extends.

  • Reutilização de Código: O código escrito na superclasse pode ser reutilizado na subclasse, evitando duplicação.
  • Extensibilidade: Permite que novas funcionalidades sejam adicionadas em subclasses sem modificar a superclasse existente.
  • Polimorfismo: Facilita o polimorfismo, onde um objeto da subclasse pode ser tratado como um objeto da superclasse.
  • Hierarquia de Classes: Ajuda a organizar classes em uma estrutura hierárquica lógica, que reflete relações do mundo real.

Exemplos de Uso da Palavra-chave extends

Exemplo Básico: Herança de Atributos e Métodos

Neste exemplo, a classe Cachorro herda atributos e métodos da classe Animal. 


// Superclasse (Classe Pai)
class Animal {
    String nome;
    int idade;

    public Animal(String nome, int idade) {
        this.nome = nome;
        this.idade = idade;
    }

    public void comer() {
        System.out.println(nome + " está comendo.");
    }

    public void dormir() {
        System.out.println(nome + " está dormindo.");
    }
}

// Subclasse (Classe Filha) que estende Animal
class Cachorro extends Animal { // Cachorro herda de Animal
    String raca;

    public Cachorro(String nome, int idade, String raca) {
        super(nome, idade); // Chama o construtor da superclasse
        this.raca = raca;
    }

    public void latir() {
        System.out.println(nome + " está latindo: Au Au!");
    }
}

public class Main {
    public static void main(String[] args) {
        Cachorro meuCachorro = new Cachorro("Buddy", 2, "Golden Retriever");

        // Atributos herdados
        System.out.println("Nome: " + meuCachorro.nome);
        System.out.println("Idade: " + meuCachorro.idade);
        System.out.println("Raça: " + meuCachorro.raca);

        // Métodos herdados
        meuCachorro.comer();
        meuCachorro.dormir();

        // Método próprio da subclasse
        meuCachorro.latir();
        // Saída:
        // Nome: Buddy
        // Idade: 2
        // Raça: Golden Retriever
        // Buddy está comendo.
        // Buddy está dormindo.
        // Buddy está latindo: Au Au!
    }
}

Exemplo: Sobrescrita de Métodos com @Override

As subclasses podem fornecer uma implementação específica para um método que já existe na superclasse, um processo conhecido como **sobrescrita de método**. A anotação @Override é opcional, mas recomendada para indicar que um método está sobrescrevendo outro. 


class Forma {
    public void desenhar() {
        System.out.println("Desenhando uma forma genérica.");
    }
}

class Circulo extends Forma {
    @Override // Indica que este método sobrescreve um método da superclasse
    public void desenhar() {
        System.out.println("Desenhando um círculo.");
    }
}

class Retangulo extends Forma {
    @Override
    public void desenhar() {
        System.out.println("Desenhando um retângulo.");
    }
}

public class Main {
    public static void main(String[] args) {
        Forma forma1 = new Forma();
        Forma forma2 = new Circulo();   // Polimorfismo
        Forma forma3 = new Retangulo(); // Polimorfismo

        forma1.desenhar(); // Saída: Desenhando uma forma genérica.
        forma2.desenhar(); // Saída: Desenhando um círculo.
        forma3.desenhar(); // Saída: Desenhando um retângulo.
    }
}

Limitações da Palavra-chave extends

  • Herança Única: Em Java, uma classe pode `extends` apenas uma superclasse. Não é permitida a herança múltipla de classes.
  • Membros private: Atributos e métodos declarados como private na superclasse não são herdados e, portanto, não são diretamente acessíveis nas subclasses. O acesso deve ser feito através de métodos públicos ou protegidos da superclasse.
  • Construtores não são Herdados: Os construtores da superclasse não são herdados pelas subclasses, mas podem ser chamados usando a palavra-chave super().

A palavra-chave extends é a base para construir hierarquias de classes robustas e reutilizáveis em Java!

Herança em Java: Reutilização de Características e Comportamentos

Descubra a herança em Java, um poderoso mecanismo de POO para criar hierarquias de classes e promover a reutilização de código.

O que é Herança?

A **herança** é um dos pilares da Programação Orientada a Objetos (POO) em Java. Ela permite que uma classe (subclasse ou classe filha) herde atributos (variáveis de instância) e métodos de outra classe (superclasse ou classe pai). Isso estabelece uma relação "é-um" (is-a), onde a subclasse é um tipo especializado da superclasse.

O principal objetivo da herança é promover a **reutilização de código**, permitindo que você defina funcionalidades comuns em uma superclasse e as utilize em múltiplas subclasses, evitando a duplicação.

Como a Herança Funciona em Java (extends)

Em Java, a herança é implementada usando a palavra-chave extends. Uma classe pode estender apenas uma outra classe (Java não suporta herança múltipla de classes, mas sim de interfaces). 


// Superclasse (Classe Pai)
class Animal {
    String nome;
    int idade;

    public Animal(String nome, int idade) {
        this.nome = nome;
        this.idade = idade;
    }

    public void comer() {
        System.out.println(nome + " está comendo.");
    }

    public void dormir() {
        System.out.println(nome + " está dormindo.");
    }
}

// Subclasse (Classe Filha) que herda de Animal
class Cachorro extends Animal {
    String raca;

    public Cachorro(String nome, int idade, String raca) {
        // Chama o construtor da superclasse
        super(nome, idade);
        this.raca = raca;
    }

    public void latir() {
        System.out.println(nome + " está latindo: Au Au!");
    }
}

public class Main {
    public static void main(String[] args) {
        Cachorro meuCachorro = new Cachorro("Rex", 3, "Labrador");

        // Acessando atributos herdados da superclasse Animal
        System.out.println("Nome do cachorro: " + meuCachorro.nome);   // Herdado
        System.out.println("Idade do cachorro: " + meuCachorro.idade); // Herdado
        System.out.println("Raça do cachorro: " + meuCachorro.raca);   // Próprio da subclasse

        // Chamando métodos herdados da superclasse Animal
        meuCachorro.comer();  // Herdado
        meuCachorro.dormir(); // Herdado

        // Chamando método próprio da subclasse Cachorro
        meuCachorro.latir();
        // Saída:
        // Nome do cachorro: Rex
        // Idade do cachorro: 3
        // Raça do cachorro: Labrador
        // Rex está comendo.
        // Rex está dormindo.
        // Rex está latindo: Au Au!
    }
}

Vantagens da Herança

  • Reutilização de Código: O maior benefício. Você escreve o código comum uma vez na superclasse e as subclasses podem utilizá-lo.
  • Economia de Tempo e Esforço: Evita a necessidade de reescrever a mesma lógica em múltiplas classes.
  • Manutenibilidade Aprimorada: Alterações na lógica comum (na superclasse) são propagadas automaticamente para todas as subclasses, facilitando a manutenção e a aplicação de correções.
  • Polimorfismo: Permite que objetos de diferentes subclasses sejam tratados como objetos de sua superclasse comum, o que é fundamental para a flexibilidade do design.
  • Organização do Código: Ajuda a criar uma hierarquia lógica e bem organizada de classes, refletindo relações do mundo real (ex: "um carro é um tipo de veículo").

Palavra-chave super

A palavra-chave super é usada dentro de uma subclasse para se referir a membros (atributos e métodos) da superclasse imediata.

  • super(): Usado para chamar o construtor da superclasse. Deve ser a primeira instrução em um construtor de subclasse.
  • super.membro: Usado para acessar atributos ou métodos da superclasse que foram sobrescritos na subclasse, ou para referenciar explicitamente um membro da superclasse quando houver ambiguidade de nome. 

class Veiculo {
    String tipo;

    public Veiculo(String tipo) {
        this.tipo = tipo;
        System.out.println("Construtor Veiculo chamado.");
    }

    public void mover() {
        System.out.println("Veículo do tipo " + tipo + " está se movendo.");
    }
}

class Carro extends Veiculo {
    String modelo;

    public Carro(String tipo, String modelo) {
        super(tipo); // Chama o construtor da superclasse Veiculo
        this.modelo = modelo;
        System.out.println("Construtor Carro chamado.");
    }

    // Sobrescrita do método mover()
    @Override // Anotação opcional, mas boa prática para indicar sobrescrita
    public void mover() {
        super.mover(); // Chama o método mover() da superclasse
        System.out.println("O carro " + modelo + " está acelerando.");
    }
}

public class Main {
    public static void main(String[] args) {
        Carro meuCarro = new Carro("terrestre", "Sedan");

        meuCarro.mover(); // Chama o método mover() sobrescrito na classe Carro
        // Saída:
        // Construtor Veiculo chamado.
        // Construtor Carro chamado.
        // Veículo do tipo terrestre está se movendo.
        // O carro Sedan está acelerando.
    }
}

Limitações e Considerações

  • Herança Única: Java não suporta herança múltipla de classes (uma classe pode herdar de apenas uma superclasse).
  • Acesso a Membros private: Membros private de uma superclasse não são herdados e, portanto, não são diretamente acessíveis nas subclasses. O acesso deve ser feito através de métodos públicos ou protegidos da superclasse.
  • Hierarquias Complexas: O uso excessivo e inadequado da herança pode levar a hierarquias de classes complexas e difíceis de manter. Em alguns casos, a composição (ter uma instância de outra classe como atributo) pode ser uma alternativa melhor.

A herança é um conceito poderoso em Java que permite a criação de código mais organizado, reutilizável e flexível!

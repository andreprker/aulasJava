Sobrescrita de Métodos (@Override) em Java

Aprenda sobre a sobrescrita de métodos em Java, um conceito fundamental do polimorfismo que permite especializar comportamentos em subclasses.

O que é Sobrescrita de Métodos (Method Overriding)?

A **sobrescrita de métodos** (Method Overriding) em Java é uma funcionalidade da Programação Orientada a Objetos (POO) que permite a uma subclasse fornecer uma implementação específica para um método que já é definido na sua superclasse. Quando um método é sobrescrito, a versão na subclasse é executada em vez da versão na superclasse, no momento da execução.

Isso é um exemplo de **polimorfismo em tempo de execução** (ou polimorfismo dinâmico), pois a decisão de qual método executar é feita pela Java Virtual Machine (JVM) durante a execução do programa, com base no tipo real do objeto.

A anotação @Override é opcional, mas fortemente recomendada. Ela informa ao compilador que o método abaixo se destina a sobrescrever um método de sua superclasse. Se você cometer um erro na assinatura do método (por exemplo, nome ou parâmetros diferentes), o compilador gerará um erro, ajudando a evitar bugs sutis.

Regras para a Sobrescrita de Métodos

  • Mesma Assinatura: O método na subclasse deve ter o mesmo nome, o mesmo número e tipo de parâmetros e a mesma ordem de parâmetros que o método na superclasse.
  • Mesmo Tipo de Retorno (ou Covariante): O tipo de retorno do método sobrescrito deve ser o mesmo ou um subtipo (covariante) do tipo de retorno do método na superclasse.
  • Nível de Acesso: O nível de acesso do método sobrescrito na subclasse não pode ser mais restritivo do que o da superclasse (ex: se o método na superclasse é protected, na subclasse ele pode ser protected ou public, mas não private).
  • Métodos final e static não podem ser sobrescritos: Métodos declarados como final não podem ser sobrescritos. Métodos static não são sobrescritos, mas sim "ocultados" (method hiding).
  • Construtores não são sobrescritos: Construtores não são herdados e, portanto, não podem ser sobrescritos.
  • Exceções: O método sobrescrito não pode lançar exceções verificadas mais amplas do que o método na superclasse.

Por que usar Sobrescrita de Métodos?

  • Especialização de Comportamento: Permite que subclasses forneçam implementações específicas para métodos herdados, adaptando o comportamento genérico da superclasse às suas necessidades particulares.
  • Polimorfismo: É essencial para o polimorfismo em tempo de execução, permitindo que objetos de diferentes subclasses respondam a uma mesma chamada de método de maneiras distintas. Isso torna o código mais genível e flexível.
  • Extensibilidade: Facilita a criação de sistemas modulares e extensíveis. Você pode adicionar novas subclasses com comportamentos especializados sem alterar o código da superclasse ou o código cliente existente que interage com a superclasse.
  • Manutenibilidade: Centraliza a lógica específica de cada tipo em sua respectiva classe, tornando o código mais fácil de entender, depurar e manter.

Exemplos de Sobrescrita de Métodos

Exemplo Básico: Sobrescrita de Comportamento

Aqui, a classe Carro e Moto sobrescrevem o método ligar() da superclasse Veiculo para fornecer suas próprias implementações. 


// Superclasse
class Veiculo {
    public void ligar() {
        System.out.println("Veículo genérico está ligando.");
    }
}

// Subclasse Carro
class Carro extends Veiculo {
    @Override // Boa prática para indicar que o método está sendo sobrescrito
    public void ligar() {
        System.out.println("Carro está ligando com a ignição.");
    }
}

// Subclasse Moto
class Moto extends Veiculo {
    @Override
    public void ligar() {
        System.out.println("Moto está ligando com o pedal de partida.");
    }
}

public class Main {
    public static void main(String[] args) {
        Veiculo veiculo1 = new Veiculo(); // Objeto da superclasse
        Veiculo veiculo2 = new Carro();   // Polimorfismo: Referência Veiculo, Objeto Carro
        Veiculo veiculo3 = new Moto();    // Polimorfismo: Referência Veiculo, Objeto Moto

        veiculo1.ligar(); // Saída: Veículo genérico está ligando.
        veiculo2.ligar(); // Saída: Carro está ligando com a ignição. (Método de Carro é chamado)
        veiculo3.ligar(); // Saída: Moto está ligando com o pedal de partida. (Método de Moto é chamado)
    }
}

Chamando o Método da Superclasse com super

Você pode usar a palavra-chave super para chamar a implementação do método da superclasse dentro do método sobrescrito na subclasse. Isso é útil para estender o comportamento em vez de substituí-lo completamente. 


class Animal {
    public void comer() {
        System.out.println("Animal está comendo.");
    }
}

class Cachorro extends Animal {
    @Override
    public void comer() {
        super.comer(); // Chama o método comer() da superclasse Animal
        System.out.println("Cachorro está comendo ração para cães.");
    }

    public void latir() {
        System.out.println("Au au!");
    }
}

public class Main {
    public static void main(String[] args) {
        Cachorro meuCachorro = new Cachorro();
        meuCachorro.comer();
        // Saída:
        // Animal está comendo.
        // Cachorro está comendo ração para cães.
    }
}
```

A sobrescrita de métodos é uma ferramenta essencial para criar hierarquias de classes flexíveis e robustas em Java, aproveitando todo o poder do polimorfismo!

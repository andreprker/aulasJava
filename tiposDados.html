Linguagem de Programação Orientada a Objetos: Tipos de Dados em Java

Em Programação Orientada a Objetos (POO), a forma como os dados são representados e manipulados é fundamental. Em Java, uma linguagem puramente orientada a objetos (com algumas exceções para tipos primitivos), entendemos os dados através de duas categorias principais: tipos de dados nativos (ou primitivos) e tipos de dados definidos pelo usuário.

Tipos de Dados Nativos (Primitivos)

Os tipos de dados nativos, também conhecidos como primitivos, são os blocos de construção mais básicos em Java. Eles não são objetos e, portanto, não possuem métodos. Seu propósito é armazenar valores simples e diretos de forma eficiente na memória. Java possui oito tipos primitivos:

  • byte: 8 bits, para números inteiros pequenos (de -128 a 127).
  • short: 16 bits, para números inteiros (de -32.768 a 32.767).
  • int: 32 bits, o tipo mais comum para números inteiros (aproximadamente de -2 bilhões a 2 bilhões).
  • long: 64 bits, para números inteiros muito grandes.
  • float: 32 bits, para números de ponto flutuante de precisão simples.
  • double: 64 bits, o tipo mais comum para números de ponto flutuante de precisão dupla.
  • boolean: 1 bit, para valores lógicos (true ou false).
  • char: 16 bits, para um único caractere Unicode.

Exemplo de uso de tipos primitivos:

int idade = 30;
double preco = 99.99;
boolean isAtivo = true;
char inicial = 'J';

Tipos de Dados Definidos pelo Usuário

Em contraste com os tipos primitivos, os tipos de dados definidos pelo usuário são o coração da Programação Orientada a Objetos. Eles permitem que os programadores criem estruturas de dados complexas e personalizadas que representam entidades do mundo real. Em Java, isso é feito principalmente através de:

1. Classes

Uma classe é um modelo ou "planta" para criar objetos. Ela define as características (atributos ou variáveis de instância) e os comportamentos (métodos) que os objetos desse tipo terão. Quando você cria uma classe, está definindo um novo tipo de dado.

Exemplo de uma classe (tipo de dado definido pelo usuário):

public class Carro {
    // Atributos (características)
    String marca;
    String modelo;
    int ano;
    String cor;

    // Construtor
    public Carro(String marca, String modelo, int ano, String cor) {
        this.marca = marca;
        this.modelo = modelo;
        this.ano = ano;
        this.cor = cor;
    }

    // Métodos (comportamentos)
    public void ligar() {
        System.out.println("O carro " + modelo + " está ligado.");
    }

    public void acelerar() {
        System.out.println("O carro está acelerando.");
    }
}

A partir dessa classe, podemos criar objetos (instâncias) de Carro:

// Criando objetos do tipo Carro
Carro meuCarro = new Carro("Toyota", "Corolla", 2022, "Prata");
Carro seuCarro = new Carro("Honda", "Civic", 2023, "Preto");

// Acessando atributos e chamando métodos
System.out.println("Meu carro é um " + meuCarro.marca + " " + meuCarro.modelo);
meuCarro.ligar();
seuCarro.acelerar();

2. Interfaces

Uma interface em Java é um contrato que especifica um conjunto de métodos que uma classe deve implementar. Ela define um comportamento comum que diferentes classes podem compartilhar, sem se preocupar com os detalhes da implementação.

Exemplo de uma interface:

public interface FormaGeometrica {
    double calcularArea();
    double calcularPerimetro();
}

public class Circulo implements FormaGeometrica {
    double raio;

    public Circulo(double raio) {
        this.raio = raio;
    }

    @Override
    public double calcularArea() {
        return Math.PI * raio * raio;
    }

    @Override
    public double calcularPerimetro() {
        return 2 * Math.PI * raio;
    }
}

public class Retangulo implements FormaGeometrica {
    double largura;
    double altura;

    public Retangulo(double largura, double altura) {
        this.largura = largura;
        this.altura = altura;
    }

    @Override
    public double calcularArea() {
        return largura * altura;
    }

    @Override
    public double calcularPerimetro() {
        return 2 * (largura + altura);
    }
}

Usando a interface para polimorfismo:

FormaGeometrica circulo1 = new Circulo(5.0);
FormaGeometrica retangulo1 = new Retangulo(4.0, 6.0);

System.out.println("Área do círculo: " + circulo1.calcularArea());
System.out.println("Perímetro do retângulo: " + retangulo1.calcularPerimetro());

3. Enums (Enumerações)

Enums são um tipo especial de classe que representa um grupo fixo de constantes. São úteis para definir uma coleção de valores relacionados e pré-definidos.

Exemplo de um Enum:

public enum DiasDaSemana {
    SEGUNDA, TERCA, QUARTA, QUINTA, SEXTA, SABADO, DOMINGO
}

public class TesteEnum {
    public static void main(String[] args) {
        DiasDaSemana hoje = DiasDaSemana.QUARTA;
        System.out.println("Hoje é: " + hoje);

        if (hoje == DiasDaSemana.QUARTA) {
            System.out.println("Estamos no meio da semana!");
        }
    }
}

Ponteiro: Variáveis de Referência

É importante notar que, em Java, quando trabalhamos com tipos de dados definidos pelo usuário (objetos de classes, interfaces, etc.), as variáveis que os armazenam não contêm o objeto em si, mas sim uma referência (um "ponteiro") para o local na memória onde o objeto está armazenado. Isso é diferente de C/C++, onde você manipula ponteiros diretamente. Em Java, a manipulação de referências é mais abstrata e segura.

Carro meuCarro = new Carro("Fiat", "Uno", 2010, "Azul"); // meuCarro é uma referência
Carro outroCarro = meuCarro; // outroCarro agora referencia o MESMO objeto que meuCarro

Neste exemplo, meuCarro e outroCarro não são cópias independentes; eles apontam para o mesmo objeto Carro na memória.

Conclusão

A distinção entre tipos de dados nativos e definidos pelo usuário é um pilar fundamental da Programação Orientada a Objetos em Java. Enquanto os tipos primitivos fornecem uma base eficiente para dados simples, as classes, interfaces e enums permitem a criação de modelos complexos e flexíveis, essenciais para construir aplicações robustas e escaláveis, promovendo a reutilização de código e a organização.

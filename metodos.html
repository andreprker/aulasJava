Métodos: Comportamentos do Objeto em Java

Na Programação Orientada a Objetos (POO), além de possuírem características (atributos que definem seu estado), os objetos também realizam ações ou executam tarefas. Essas ações são definidas pelos métodos. Enquanto os atributos representam o que um objeto é, os métodos definem o que um objeto pode fazer.

---

O que são Métodos em Java?

Métodos são blocos de código que contêm uma série de instruções para executar uma operação específica. Eles são declarados dentro de uma classe e podem ser invocados (chamados) para realizar suas funções. Cada método representa um comportamento ou uma funcionalidade que os objetos dessa classe podem exibir.

Por exemplo, se tivermos uma classe Carro, ela pode ter métodos como ligar(), acelerar(), frear() e desligar(). Cada vez que você invoca um desses métodos em um objeto Carro, ele executa a ação correspondente.

---

Estrutura de um Método

A estrutura básica de um método em Java inclui:

  • Modificador de Acesso: (public, private, protected, padrão/package-private) Define a visibilidade do método.
  • Tipo de Retorno: O tipo de dado que o método retornará (ou void se não retornar nada).
  • Nome do Método: Um identificador único para o método, seguindo as convenções de nomenclatura Java (camelCase).
  • Lista de Parâmetros: Uma lista de variáveis que o método aceita como entrada, entre parênteses (vazia se não aceitar parâmetros).
  • Corpo do Método: O bloco de código que contém as instruções a serem executadas, delimitado por chaves {}.
// Exemplo de estrutura de um método
public int somar(int numero1, int numero2) {
    // Corpo do método
    int resultado = numero1 + numero2;
    return resultado; // Retorna o valor inteiro
}
---

Definindo Comportamentos do Objeto

Os métodos são a forma como interagimos com o estado de um objeto. Eles podem:

  • Modificar o estado interno do objeto: Alterar os valores dos atributos do objeto (ex: acelerar() que aumenta a velocidadeAtual de um Carro).
  • Acessar o estado interno do objeto: Ler os valores dos atributos do objeto (ex: getVelocidadeAtual()).
  • Realizar cálculos ou operações: Processar dados e retornar um resultado (ex: calcularArea() em uma classe Retangulo).
  • Interagir com outros objetos: Chamar métodos de outros objetos ou passar este objeto como parâmetro.

Exemplo Prático em Java

Vamos estender a classe Livro do exemplo anterior para incluir mais métodos e ilustrar seus comportamentos.

Classe 'Livro.java' (Atualizada)

public class Livro {
    // Atributos (variáveis de instância)
    String titulo;
    String autor;
    int anoPublicacao;
    boolean disponivel;
    int numeroCopias; // Novo atributo: quantidade de cópias disponíveis

    // Construtor para inicializar o estado do objeto
    public Livro(String titulo, String autor, int anoPublicacao, int numeroCopias) {
        this.titulo = titulo;
        this.autor = autor;
        this.anoPublicacao = anoPublicacao;
        this.numeroCopias = numeroCopias;
        this.disponivel = (numeroCopias > 0); // O livro está disponível se houver cópias
    }

    // Método para exibir o estado do livro
    public void exibirDetalhes() {
        System.out.println("--- Detalhes do Livro ---");
        System.out.println("Título: " + titulo);
        System.out.println("Autor: " + autor);
        System.out.println("Ano de Publicação: " + anoPublicacao);
        System.out.println("Cópias Disponíveis: " + numeroCopias);
        System.out.println("Disponível para Empréstimo: " + (disponivel ? "Sim" : "Não"));
    }

    // Método para mudar o estado de disponibilidade e o número de cópias
    public boolean emprestarLivro() {
        if (disponivel && numeroCopias > 0) {
            this.numeroCopias--;
            this.disponivel = (this.numeroCopias > 0); // Atualiza disponibilidade
            System.out.println("Livro '" + titulo + "' emprestado. Cópias restantes: " + numeroCopias);
            return true;
        } else {
            System.out.println("Livro '" + titulo + "' não pode ser emprestado (sem cópias ou já indisponível).");
            return false;
        }
    }

    public void devolverLivro() {
        this.numeroCopias++;
        this.disponivel = true; // Ao devolver, ele fica disponível
        System.out.println("Livro '" + titulo + "' devolvido. Cópias totais: " + numeroCopias);
    }

    // Novo método para verificar se o livro é antigo
    public boolean ehAntigo(int anoAtual) {
        return (anoAtual - this.anoPublicacao) > 50; // Considera antigo se publicado há mais de 50 anos
    }

    // Getter para o título (acessa o estado)
    public String getTitulo() {
        return this.titulo;
    }
}

Classe Principal para Teste ('Main.java') (Atualizada)

public class Main {
    public static void main(String[] args) {
        // Criando instâncias (objetos) da classe Livro
        Livro livro1 = new Livro("A Revolução dos Bichos", "George Orwell", 1945, 2);
        Livro livro2 = new Livro("O Pequeno Príncipe", "Antoine de Saint-Exupéry", 1943, 0); // Sem cópias inicialmente

        System.out.println("--- Estado Inicial dos Livros ---");
        livro1.exibirDetalhes();
        System.out.println();
        livro2.exibirDetalhes();

        System.out.println("\n--- Interagindo com Livro 1 ---");
        livro1.emprestarLivro(); // Comportamento: emprestar
        livro1.exibirDetalhes();

        livro1.emprestarLivro(); // Empresta mais uma cópia
        livro1.exibirDetalhes();

        livro1.emprestarLivro(); // Tenta emprestar quando não há mais cópias
        livro1.exibirDetalhes();

        livro1.devolverLivro(); // Comportamento: devolver
        livro1.exibirDetalhes();

        System.out.println("\n--- Interagindo com Livro 2 ---");
        livro2.emprestarLivro(); // Tenta emprestar um livro sem cópias
        livro2.exibirDetalhes();

        // Verificando comportamentos baseados no estado
        System.out.println("\n--- Verificando se os livros são antigos ---");
        int anoAtual = 2025;
        if (livro1.ehAntigo(anoAtual)) { // Comportamento: verificar antiguidade
            System.out.println("O livro '" + livro1.getTitulo() + "' é considerado antigo.");
        } else {
            System.out.println("O livro '" + livro1.getTitulo() + "' não é considerado antigo.");
        }

        if (livro2.ehAntigo(anoAtual)) {
            System.out.println("O livro '" + livro2.getTitulo() + "' é considerado antigo.");
        } else {
            System.out.println("O livro '" + livro2.getTitulo() + "' não é considerado antigo.");
        }
    }
}

Relação entre Atributos e Métodos

Atributos e métodos trabalham em conjunto para definir completamente um objeto. Os atributos guardam o estado, e os métodos são as "ações" que podem ler, modificar ou usar esse estado para produzir novos comportamentos ou informações. Essa colaboração é a essência do encapsulamento na POO.

Conclusão

Métodos são os construtores dos comportamentos em Java, permitindo que os objetos realizem tarefas e interajam com o mundo exterior. Eles são cruciais para a funcionalidade de uma aplicação orientada a objetos, traduzindo os atributos estáticos em ações dinâmicas. Dominar a criação e o uso de métodos é essencial para qualquer desenvolvedor Java.

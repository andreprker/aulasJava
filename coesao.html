Coesão (Cohesion): Foco de Responsabilidade de uma Classe em Java

Entenda o que é coesão em Java, seus diferentes tipos e por que é essencial buscar uma alta coesão para um design de software robusto e manutenível.

O que é Coesão?

Em engenharia de software, **coesão** (ou *cohesion*) refere-se ao grau em que os elementos de um módulo (como uma classe) pertencem uns aos outros. É uma medida de quão bem os atributos e métodos de uma classe trabalham juntos para fornecer uma única e bem definida funcionalidade.

Uma **alta coesão** significa que a classe tem um propósito bem focado e todas as suas partes estão diretamente relacionadas a esse propósito. Por outro lado, uma **baixa coesão** indica que a classe tem responsabilidades múltiplas, não relacionadas, ou que seus elementos internos não estão logicamente conectados, o que é indesejável.

Por que a Coesão é Importante? (Benefícios da Alta Coesão)

  • Compreensibilidade: Classes com alta coesão são mais fáceis de entender, pois sua funcionalidade é clara e bem definida. É mais fácil saber o que a classe faz e como ela deve ser usada.
  • Manutenibilidade: Alterações em uma classe com alta coesão são menos propensas a afetar outras partes do sistema, pois suas responsabilidades são isoladas. Isso reduz a complexidade da manutenção e a chance de introduzir novos bugs.
  • Reusabilidade: Classes bem coesas são mais fáceis de serem reutilizadas em diferentes contextos, pois representam uma unidade lógica e funcional independente.
  • Testabilidade: Componentes com alta coesão são mais fáceis de testar isoladamente, pois possuem um foco claro e menos dependências externas desnecessárias.
  • Flexibilidade: Um sistema com componentes de alta coesão é mais flexível e adaptável a mudanças, pois as modificações em uma funcionalidade específica são contidas dentro da classe responsável.

Tipos de Coesão (do pior para o melhor)

Existem vários tipos de coesão, variando do mais fraco (indesejável) ao mais forte (desejável):

1. Coesão Coincidental (Coincidental Cohesion)

Considerado o pior tipo de coesão. Os elementos de um módulo são agrupados aleatoriamente, sem nenhuma relação lógica significativa. É como agrupar funcionalidades diversas em uma única classe porque "pareceu conveniente".

2. Coesão Lógica (Logical Cohesion)

Os elementos de um módulo são agrupados porque executam funções logicamente semelhantes, mas as funcionalidades não estão diretamente relacionadas (ex: uma classe que tem métodos para "printar todos os relatórios", "printar um relatório de vendas", "printar um relatório de estoque", onde a lógica de cada impressão é distinta). A classe decide qual lógica executar com base em um flag ou tipo.

3. Coesão Temporal (Temporal Cohesion)

Os elementos de um módulo são agrupados porque são executados juntos em um determinado momento (ex: um método "inicializarSistema" que faz várias coisas não relacionadas, como abrir arquivos de log, conectar ao banco de dados e carregar configurações).

4. Coesão Procedural (Procedural Cohesion)

Os elementos de um módulo são agrupados porque participam de uma sequência de operações. A saída de um elemento se torna a entrada de outro (ex: uma função que lê dados, processa e depois os salva). Melhor que os anteriores, mas ainda pode agrupar responsabilidades diferentes.

5. Coesão Comunicacional (Communicational Cohesion)

Os elementos de um módulo são agrupados porque operam sobre os mesmos dados de entrada ou produzem os mesmos dados de saída (ex: uma classe que tem métodos para calcular o preço de um produto e aplicar um desconto, ambos usando o mesmo objeto `Produto`).

6. Coesão Funcional (Functional Cohesion)

Considerado o tipo de coesão mais forte e desejável. O módulo executa uma única função bem definida. Todos os elementos são essenciais para a execução dessa única função. A classe tem uma única responsabilidade.

Exemplo Prático (Alta Coesão vs. Baixa Coesão)

Exemplo de Baixa Coesão (EVITAR!)

A classe GerenciadorGeral tem múltiplas responsabilidades não relacionadas, como gerenciar usuários, processar pagamentos e gerar relatórios. Uma mudança em uma dessas áreas afetará a classe inteira. 


// Exemplo de Baixa Coesão
class GerenciadorGeral {
    public void criarUsuario(String nome) {
        System.out.println("Usuário " + nome + " criado.");
        // Lógica de banco de dados, validação, etc.
    }

    public void processarPagamento(double valor) {
        System.out.println("Pagamento de " + valor + " processado.");
        // Lógica de gateway de pagamento, notificação, etc.
    }

    public void gerarRelatorioVendas() {
        System.out.println("Gerando relatório de vendas...");
        // Lógica de consulta ao banco de dados, formatação, etc.
    }

    public void enviarEmailConfirmacao(String email) {
        System.out.println("Email de confirmação enviado para " + email);
        // Lógica de envio de email
    }
}

public class MainBaixaCoesao {
    public static void main(String[] args) {
        GerenciadorGeral gerenciador = new GerenciadorGeral();
        gerenciador.criarUsuario("Alice");
        gerenciador.processarPagamento(100.0);
        gerenciador.gerarRelatorioVendas();
        gerenciador.enviarEmailConfirmacao("alice@example.com");
    }
}

Exemplo de Alta Coesão (DESEJÁVEL!)

Aqui, as responsabilidades foram divididas em classes separadas, cada uma com um foco único. Isso promove a **alta coesão** e o **Princípio da Responsabilidade Única (SRP)**. 


// Exemplo de Alta Coesão

class GerenciadorDeUsuario {
    public void criarUsuario(String nome) {
        System.out.println("Usuário " + nome + " criado.");
        // Lógica de banco de dados, validação, etc.
    }
}

class ProcessadorDePagamento {
    public void processar(double valor) {
        System.out.println("Pagamento de " + valor + " processado.");
        // Lógica de gateway de pagamento, notificação, etc.
    }
}

class GeradorDeRelatorio {
    public void gerarRelatorioVendas() {
        System.out.println("Gerando relatório de vendas...");
        // Lógica de consulta ao banco de dados, formatação, etc.
    }
}

class EnviadorDeEmail {
    public void enviarConfirmacao(String email) {
        System.out.println("Email de confirmação enviado para " + email);
        // Lógica de envio de email
    }
}

public class MainAltaCoesao {
    public static void main(String[] args) {
        GerenciadorDeUsuario gerenciadorUsuario = new GerenciadorDeUsuario();
        ProcessadorDePagamento processadorPagamento = new ProcessadorDePagamento();
        GeradorDeRelatorio geradorRelatorio = new GeradorDeRelatorio();
        EnviadorDeEmail enviadorEmail = new EnviadorDeEmail();

        gerenciadorUsuario.criarUsuario("Bob");
        processadorPagamento.processar(250.0);
        geradorRelatorio.gerarRelatorioVendas();
        enviadorEmail.enviarConfirmacao("bob@example.com");
    }
}

Buscar a alta coesão é uma prática fundamental para criar software Java mais robusto, flexível, compreensível e fácil de manter!

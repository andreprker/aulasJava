Interfaces: Contratos de Métodos em Java

Descubra como interfaces em Java atuam como contratos, definindo comportamentos que as classes devem implementar, essenciais para abstração e polimorfismo.

O que são Interfaces?

Em Java, uma **interface** é um tipo de referência que define um conjunto de métodos (e, opcionalmente, constantes, métodos padrão, métodos estáticos e métodos privados). No entanto, e este é o ponto crucial, ela não fornece a implementação desses métodos.

Pense em uma interface como um **contrato**. Quando uma classe implementa uma interface, ela se compromete a fornecer uma implementação concreta para todos os métodos abstratos declarados nessa interface. É uma maneira poderosa de alcançar a abstração total e o polimorfismo em Java.

Interfaces são declaradas usando a palavra-chave interface.

Por que usar Interfaces? (Benefícios)

  • Abstração Completa: Interfaces definem o "o quê" sem especificar o "como". Elas permitem que você modele comportamentos sem se preocupar com os detalhes de implementação.
  • Herança Múltipla de Tipo: Embora Java não suporte herança múltipla de classes, uma classe pode implementar múltiplas interfaces. Isso permite que uma classe herde comportamentos de diferentes "contratos", adicionando flexibilidade ao design.
  • Acoplamento Fraco (Loose Coupling): Interfaces promovem um design onde componentes interagem através de uma interface, em vez de depender diretamente de implementações concretas. Isso torna o sistema mais flexível e fácil de modificar.
  • Polimorfismo: Interfaces são a base do polimorfismo em tempo de execução. Uma referência do tipo de interface pode apontar para qualquer objeto de uma classe que a implementa, permitindo que você processe objetos de diferentes tipos de forma uniforme.
  • Design de APIs: São amplamente utilizadas para definir APIs (Application Programming Interfaces) em bibliotecas e frameworks, fornecendo um conjunto de regras claras para a interação.

Como Implementar Interfaces em Java

Para usar uma interface, uma classe deve implementar essa interface usando a palavra-chave implements.

Exemplo 1: Interface Simples e Uma Implementação

Aqui, a interface Desenhavel define um método, e a classe Circulo o implementa. 


// Definição da Interface
interface Desenhavel {
    // Métodos em interfaces são implicitamente public e abstract (até Java 8)
    void desenhar();
}

// Classe implementando a interface
class Circulo implements Desenhavel {
    @Override // Anotação opcional, mas boa prática
    public void desenhar() {
        System.out.println("Desenhando um círculo.");
    }
}

public class Main {
    public static void main(String[] args) {
        Circulo meuCirculo = new Circulo();
        meuCirculo.desenhar(); // Saída: Desenhando um círculo.

        // Polimorfismo: Referência do tipo da interface, objeto do tipo da classe
        Desenhavel forma = new Circulo();
        forma.desenhar(); // Saída: Desenhando um círculo.
    }
}

Exemplo 2: Implementando Múltiplas Interfaces

Uma classe pode implementar várias interfaces, desde que forneça a implementação para todos os métodos de todas as interfaces. 


interface Voavel {
    void voar();
}

interface Nadavel {
    void nadar();
}

class Pato implements Voavel, Nadavel { // Pato implementa ambas as interfaces
    @Override
    public void voar() {
        System.out.println("Pato está voando.");
    }

    @Override
    public void nadar() {
        System.out.println("Pato está nadando.");
    }

    public void grasnar() {
        System.out.println("Pato faz quack!");
    }
}

public class Main {
    public static void main(String[] args) {
        Pato meuPato = new Pato();
        meuPato.voar();    // Saída: Pato está voando.
        meuPato.nadar();   // Saída: Pato está nadando.
        meuPato.grasnar(); // Saída: Pato faz quack!

        Voavel serVoador = new Pato(); // Polimorfismo
        serVoador.voar(); // Saída: Pato está voando.

        Nadavel serNadador = new Pato(); // Polimorfismo
        serNadador.nadar(); // Saída: Pato está nadando.
    }
}

Exemplo 3: Constantes e Métodos Padrão em Interfaces (Java 8+)

A partir do Java 8, interfaces podem ter:

  • **Constantes:** Variáveis são implicitamente public static final.
  • **Métodos Padrão (default):** Métodos com implementação padrão, que as classes que implementam a interface podem optar por sobrescrever ou usar a implementação padrão.
  • **Métodos Estáticos:** Métodos que pertencem à interface em si e podem ser chamados diretamente pela interface.


interface OperacaoMatematica {
    // Constante (implicitamente public static final)
    double PI = 3.14159;

    // Método abstrato (implicitamente public abstract)
    double calcular(double a, double b);

    // Método padrão (default method)
    default void exibirMensagem() {
        System.out.println("Realizando uma operação matemática.");
    }

    // Método estático
    static double dobrar(double valor) {
        return valor * 2;
    }
}

class Soma implements OperacaoMatematica {
    @Override
    public double calcular(double a, double b) {
        return a + b;
    }

    // Opcionalmente, pode sobrescrever o método padrão
    @Override
    public void exibirMensagem() {
        System.out.println("Realizando uma operação de SOMA.");
    }
}

class Subtracao implements OperacaoMatematica {
    @Override
    public double calcular(double a, double b) {
        return a - b;
    }
}

public class Main {
    public static void main(String[] args) {
        Soma soma = new Soma();
        Subtracao subtracao = new Subtracao();

        System.out.println("Soma de 10 e 5: " + soma.calcular(10, 5)); // Saída: Soma de 10 e 5: 15.0
        soma.exibirMensagem(); // Saída: Realizando uma operação de SOMA.

        System.out.println("Subtração de 10 e 5: " + subtracao.calcular(10, 5)); // Saída: Subtração de 10 e 5: 5.0
        subtracao.exibirMensagem(); // Saída: Realizando uma operação matemática. (usa o default)

        System.out.println("Valor de PI: " + OperacaoMatematica.PI); // Acessando constante
        System.out.println("Dobro de 7: " + OperacaoMatematica.dobrar(7)); // Chamando método estático
        // Saída:
        // Soma de 10 e 5: 15.0
        // Realizando uma operação de SOMA.
        // Subtração de 10 e 5: 5.0
        // Realizando uma operação matemática.
        // Valor de PI: 3.14159
        // Dobro de 7: 14.0
    }
}
```

Regras Chave para Interfaces

  • Todos os métodos em uma interface são implicitamente public e abstract (antes do Java 8).
  • Atributos em interfaces são implicitamente public static final (constantes).
  • Interfaces não podem ter construtores.
  • Uma classe pode implementar múltiplas interfaces.
  • Uma interface pode estender múltiplas interfaces.

Interfaces são uma ferramenta poderosa em Java para criar designs de software flexíveis, extensíveis e polimórficos!

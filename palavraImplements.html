Implementação de Interfaces: implements em Java

Descubra a palavra-chave implements em Java, essencial para classes que se comprometem a seguir os contratos definidos pelas interfaces.

O que é a Palavra-chave implements?

Em Java, a palavra-chave implements é usada na declaração de uma classe para indicar que essa classe está **fornecendo uma implementação concreta** para os métodos abstratos declarados em uma ou mais interfaces.

Quando uma classe implements uma interface, ela assume a responsabilidade de cumprir o "contrato" dessa interface. Isso significa que a classe deve fornecer um corpo (uma implementação) para cada método abstrato presente na interface. Se a classe não implementar todos os métodos, ela deverá ser declarada como uma classe abstrata.

Por que usar a Palavra-chave implements?

  • Cumprimento de Contrato: Garante que a classe forneça uma implementação para um conjunto específico de comportamentos definidos pela interface.
  • Polimorfismo: Permite que objetos de diferentes classes que implementam a mesma interface sejam tratados de forma uniforme através da referência da interface, facilitando o código genérico.
  • Herança Múltipla de Tipo: Diferente da herança de classes (com `extends`), uma classe pode implementar múltiplas interfaces, permitindo que ela possua comportamentos de "tipos" distintos.
  • Design Flexível: Promove um design de software onde os componentes interagem por meio de interfaces (contratos), o que os torna menos acoplados e mais fáceis de substituir ou modificar.
  • Separação de Preocupações: Interfaces podem definir diferentes "habilidades" ou "papéis" que uma classe pode ter, ajudando a separar responsabilidades.

Como usar a Palavra-chave implements

A sintaxe para implementar uma ou mais interfaces é a seguinte: 


// Para uma única interface
class MinhaClasse implements MinhaInterface {
    // Implementação dos métodos da interface
}

// Para múltiplas interfaces
class OutraClasse implements InterfaceUm, InterfaceDois, InterfaceTres {
    // Implementação dos métodos de todas as interfaces
}

Exemplo 1: Implementando uma Única Interface

Uma classe Automovel que implementa a interface Veiculo. 


// Interface (o contrato)
interface Veiculo {
    void acelerar();
    void frear();
}

// Classe que implementa a interface Veiculo
class Automovel implements Veiculo {
    private String marca;

    public Automovel(String marca) {
        this.marca = marca;
    }

    @Override
    public void acelerar() {
        System.out.println(marca + " está acelerando.");
    }

    @Override
    public void frear() {
        System.out.println(marca + " está freando.");
    }

    public void ligarRadio() {
        System.out.println(marca + " ligou o rádio.");
    }
}

public class Main {
    public static void main(String[] args) {
        Automovel carro = new Automovel("Ford");
        carro.acelerar();    // Saída: Ford está acelerando.
        carro.frear();       // Saída: Ford está freando.
        carro.ligarRadio();  // Saída: Ford ligou o rádio.

        // Polimorfismo: Referência do tipo da interface
        Veiculo meuVeiculo = new Automovel("Honda");
        meuVeiculo.acelerar(); // Saída: Honda está acelerando.
        meuVeiculo.frear();    // Saída: Honda está freando.
        // meuVeiculo.ligarRadio(); // ERRO DE COMPILAÇÃO: Veiculo não tem ligarRadio()
    }
}

Exemplo 2: Implementando Múltiplas Interfaces

Uma classe Smartphone que implementa as interfaces Chamavel e Navegavel. 


interface Chamavel {
    void fazerChamada(String numero);
}

interface Navegavel {
    void navegar(String url);
}

// Classe que implementa múltiplas interfaces
class Smartphone implements Chamavel, Navegavel {
    private String modelo;

    public Smartphone(String modelo) {
        this.modelo = modelo;
    }

    @Override
    public void fazerChamada(String numero) {
        System.out.println(modelo + " está ligando para: " + numero);
    }

    @Override
    public void navegar(String url) {
        System.out.println(modelo + " está navegando para: " + url);
    }

    public void tirarFoto() {
        System.out.println(modelo + " tirou uma foto.");
    }
}

public class Main {
    public static void main(String[] args) {
        Smartphone meuSmartphone = new Smartphone("Galaxy S23");
        meuSmartphone.fazerChamada("9876-5432"); // Saída: Galaxy S23 está ligando para: 9876-5432
        meuSmartphone.navegar("https://www.google.com"); // Saída: Galaxy S23 está navegando para: https://www.google.com
        meuSmartphone.tirarFoto(); // Saída: Galaxy S23 tirou uma foto.

        // Polimorfismo com múltiplas interfaces
        Chamavel dispositivoChamavel = meuSmartphone;
        dispositivoChamavel.fazerChamada("1234-5678");

        Navegavel dispositivoNavegavel = meuSmartphone;
        dispositivoNavegavel.navegar("https://www.openai.com");
        // Saída:
        // Galaxy S23 está ligando para: 1234-5678
        // Galaxy S23 está navegando para: https://www.openai.com
    }
}

Exemplo 3: Lidando com Métodos Padrão (`default`) em Interfaces (Java 8+)

Classes que implementam interfaces com métodos `default` não são obrigadas a sobrescrevê-los. Elas podem usar a implementação padrão ou fornecer a sua própria. 


interface ReprodutorDeAudio {
    void tocar(String faixa);
    void pausar();

    // Método padrão (default)
    default void ajustarVolume(int nivel) {
        System.out.println("Volume ajustado para: " + nivel);
    }
}

class MP3Player implements ReprodutorDeAudio {
    @Override
    public void tocar(String faixa) {
        System.out.println("Tocando: " + faixa + " no MP3 Player.");
    }

    @Override
    public void pausar() {
        System.out.println("Música pausada no MP3 Player.");
    }
    // Não sobrescreve ajustarVolume(), usará a implementação padrão
}

class StreamingPlayer implements ReprodutorDeAudio {
    @Override
    public void tocar(String faixa) {
        System.out.println("Streaming: " + faixa + " online.");
    }

    @Override
    public void pausar() {
        System.out.println("Streaming pausado.");
    }

    // Sobrescrevendo o método padrão para comportamento específico
    @Override
    public void ajustarVolume(int nivel) {
        System.out.println("Ajustando volume do streaming para: " + nivel + "%");
    }
}

public class Main {
    public static void main(String[] args) {
        MP3Player meuMP3 = new MP3Player();
        meuMP3.tocar("Minha Canção Favorita");
        meuMP3.ajustarVolume(7); // Usa a implementação padrão
        meuMP3.pausar();

        System.out.println("\n---");

        StreamingPlayer meuStreaming = new StreamingPlayer();
        meuStreaming.tocar("Live Concert");
        meuStreaming.ajustarVolume(10); // Usa a implementação sobrescrita
        meuStreaming.pausar();
        // Saída:
        // Tocando: Minha Canção Favorita no MP3 Player.
        // Volume ajustado para: 7
        // Música pausada no MP3 Player.

        // ---
        // Streaming: Live Concert online.
        // Ajustando volume do streaming para: 10%
        // Streaming pausado.
    }
}
```

Regras Chave para implements

  • Uma classe usa a palavra-chave implements para indicar que implementará uma interface.
  • Uma classe **deve** implementar todos os métodos abstratos da interface, a menos que a própria classe seja abstrata.
  • Uma classe pode implementar múltiplas interfaces, separadas por vírgulas.
  • Se uma classe herda de outra e implementa interfaces, a palavra-chave extends vem antes de implements (ex: class MinhaClasse extends SuperClass implements Interface1, Interface2).

A palavra-chave implements é a ponte entre as definições de comportamento das interfaces e as implementações concretas das classes em Java!

Aulas de POO em Java

Curso Prático de POO em Java

Explore os fundamentos e pilares da Programação Orientada a Objetos em Java através de aulas práticas e focadas, cada uma com 50 minutos de duração.

Aula 0: Variáveis

Esta aula mostra os tipos de dados existentes e como fazer manipulações simples com variáveis.

Conteúdo Programático (15 min)

  • Tipos de dados em Java.
  • Variáveis: como declarar, exibir e receber valores.

Aula 1: Introdução à POO, Classes e Objetos

Esta aula introduz os conceitos fundamentais da Programação Orientada a Objetos, estabelecendo a diferença crucial entre classes (moldes) e objetos (instâncias concretas).

Conteúdo Programático (15 min)

  • O que é POO e suas vantagens.
  • O "molde" para objetos.
  • Instâncias de uma classe.
  • Sintaxe básica de classe e criação de objeto.

Atividade Prática (25 min)

Criar uma classe `Pessoa` e, na classe `Main`, instanciar dois objetos `Pessoa`. Imprimir uma mensagem de confirmação no console para cada objeto criado.

public class Pessoa { }

public class Main {
    public static void main(String[] args) {
        Pessoa p1 = new Pessoa();
        Pessoa p2 = new Pessoa();
        System.out.println("Objeto p1 criado!");
        System.out.println("Objeto p2 criado!");
    }
}

Aula 2: Atributos e Métodos

Nesta aula, os alunos aprenderão a definir características (atributos) e comportamentos (métodos) para suas classes, e como interagir com eles através dos objetos.

Conteúdo Programático (15 min)

  • Atributos (variáveis de instância): Estado do objeto.
  • Métodos: Comportamentos do objeto.
  • Declaração e chamada de atributos/métodos.

Atividade Prática (25 min)

Adicionar `String nome` e `int idade` à classe `Pessoa`. Criar um método `apresentar()` que imprima os dados. Na `Main`, atribuir valores e chamar `apresentar()` para objetos.

public class Pessoa {
    String nome;
    int idade;

    public void apresentar() {
        System.out.println("Olá, meu nome é " + nome + " e tenho " + idade + " anos.");
    }
}

public class Main {
    public static void main(String[] args) {
        Pessoa p1 = new Pessoa();
        p1.nome = "Alice";
        p1.idade = 30;
        p1.apresentar();
    }
}

Aula 3: Construtores e a Palavra-Chave `this`

Esta aula aborda como inicializar objetos de forma controlada utilizando construtores e explica o uso da palavra-chave `this` para referenciar o próprio objeto.

Conteúdo Programático (15 min)

  • Construtores: Métodos especiais de inicialização.
  • Construtor padrão vs. parametrizado.
  • Sobrecarga de construtores.
  • Palavra-chave `this`: Referência ao objeto atual.

Atividade Prática (25 min)

Adicionar um construtor padrão e um parametrizado à classe `Pessoa`. Na `Main`, criar objetos usando ambos os construtores e chamar `apresentar()`.

public class Pessoa {
    String nome;
    int idade;

    public Pessoa() { // Construtor padrão
        this.nome = "Desconhecido";
        this.idade = 0;
    }

    public Pessoa(String nome, int idade) { // Construtor parametrizado
        this.nome = nome;
        this.idade = idade;
    }

    public void apresentar() {
        System.out.println("Olá, meu nome é " + nome + " e tenho " + idade + " anos.");
    }
}

public class Main {
    public static void main(String[] args) {
        Pessoa p3 = new Pessoa();
        Pessoa p4 = new Pessoa("Carlos", 25);
        p3.apresentar();
        p4.apresentar();
    }
}

Aula 4: Modificadores de Acesso (Public, Private)

Esta aula foca na compreensão e aplicação dos modificadores de acesso `public` e `private` para controlar a visibilidade e proteger os membros de uma classe.

Conteúdo Programático (15 min)

  • Controle de Acesso: Por que é importante.
  • `public`: Acessível de qualquer lugar.
  • `private`: Acessível apenas dentro da própria classe.
  • Aplicação em atributos e métodos.

Atividade Prática (25 min)

Modificar os atributos `nome` e `idade` da classe `Pessoa` para `private`. Tentar acessá-los diretamente na `Main` (observar o erro). Criar um método `public void mudarNome(String novoNome)` para alterar o nome.

public class Pessoa {
    private String nome;
    private int idade;

    public Pessoa(String nome, int idade) {
        this.nome = nome;
        this.idade = idade;
    }

    public void apresentar() {
        System.out.println("Olá, meu nome é " + nome + " e tenho " + idade + " anos.");
    }

    public void mudarNome(String novoNome) {
        this.nome = novoNome;
    }
}

public class Main {
    public static void main(String[] args) {
        Pessoa p1 = new Pessoa("Ana", 28);
        // p1.nome = "Erro"; // Isso causaria um erro de compilação
        p1.mudarNome("Anabela");
        p1.apresentar();
    }
}

Aula 5: Encapsulamento e Propriedades (Getters/Setters)

Esta aula aprofunda o conceito de encapsulamento, ensinando a implementar métodos "getter" e "setter" para acessar e modificar atributos privados de forma controlada e segura.

Conteúdo Programático (15 min)

  • Encapsulamento: Esconder detalhes de implementação.
  • Métodos Getters (Acessores): `getAtributo()`.
  • Métodos Setters (Modificadores): `setAtributo(Tipo valor)`.
  • Benefícios do encapsulamento.

Atividade Prática (25 min)

Gerar (ou escrever) métodos `getNome()`, `setNome()`, `getIdade()`, `setIdade()` para a classe `Pessoa`. Na `Main`, usar esses métodos para interagir com os objetos.

public class Pessoa {
    private String nome;
    private int idade;

    public Pessoa(String nome, int idade) {
        this.nome = nome;
        this.idade = idade;
    }

    public String getNome() { return nome; }
    public void setNome(String nome) { this.nome = nome; }

    public int getIdade() { return idade; }
    public void setIdade(int idade) {
        if (idade >= 0) { // Exemplo de validação
            this.idade = idade;
        } else {
            System.out.println("Idade inválida!");
        }
    }

    public void apresentar() {
        System.out.println("Olá, meu nome é " + nome + " e tenho " + idade + " anos.");
    }
}

public class Main {
    public static void main(String[] args) {
        Pessoa p1 = new Pessoa("Bruno", 22);
        System.out.println("Nome: " + p1.getNome());
        p1.setIdade(23);
        p1.apresentar();
    }
}

Aula 6: Herança

Esta aula introduz o conceito de herança em POO, permitindo a reutilização de código e a criação de relações "é um tipo de" entre classes, modelando hierarquias do mundo real.

Conteúdo Programático (15 min)

  • Herança: Reutilização de características e comportamentos.
  • Relação "é um tipo de" (`is-a relationship`).
  • Palavra-chave `extends`.
  • Superclasse e Subclasse.
  • Palavra-chave `super` para acessar membros da superclasse.

Atividade Prática (25 min)

Criar uma superclasse `Animal` com `nome` e `fazerSom()`. Criar uma subclasse `Cachorro` que estende `Animal`, adiciona `raca` e sobrescreve `fazerSom()`. Instanciar `Cachorro` na `Main`.

public class Animal {
    String nome;
    public void fazerSom() {
        System.out.println("Animal fazendo som...");
    }
}

public class Cachorro extends Animal {
    String raca;
    
    @Override
    public void fazerSom() {
        System.out.println("Latido!");
    }

    public void latir() {
        System.out.println("Woof woof!");
    }
}

public class Main {
    public static void main(String[] args) {
        Cachorro meuCachorro = new Cachorro();
        meuCachorro.nome = "Rex";
        meuCachorro.raca = "Labrador";
        meuCachorro.fazerSom();
        meuCachorro.latir();
    }
}

Aula 7: Polimorfismo

Esta aula explora o polimorfismo em Java, a capacidade de objetos de diferentes classes serem tratados de forma genérica através de uma superclasse ou interface, simplificando o código e aumentando a flexibilidade.

Conteúdo Programático (15 min)

  • Polimorfismo: "Muitas formas".
  • Sobrescrita de métodos (`@Override`).
  • Uso de referências de superclasse para objetos de subclasse.

Atividade Prática (25 min)

Criar uma subclasse `Gato` que estende `Animal` e sobrescreve `fazerSom()`. Na `Main`, criar um array de `Animal` e adicionar objetos `Cachorro` e `Gato`. Iterar e chamar `fazerSom()` para cada um.

public class Gato extends Animal {
    @Override
    public void fazerSom() {
        System.out.println("Miado!");
    }
    public void miar() {
        System.out.println("Miau!");
    }
}

public class Main {
    public static void main(String[] args) {
        Animal[] animais = new Animal[2];
        animais[0] = new Cachorro();
        animais[1] = new Gato();

        for (Animal a : animais) {
            a.fazerSom(); // Polimorfismo em ação
        }
    }
}

Aula 8: Interfaces

Esta aula introduz o conceito de interfaces em Java, que atuam como contratos para classes, promovendo polimorfismo e um design mais flexível e desacoplado.

Conteúdo Programático (15 min)

  • Interfaces: Contratos de métodos.
  • Palavra-chave `interface`.
  • Implementação de interfaces: `implements`.
  • Interfaces vs. Classes Abstratas (breve).

Atividade Prática (25 min)

Criar uma interface `Veiculo` com métodos `acelerar()` e `frear()`. Criar classes `Carro` e `Moto` que implementam `Veiculo`. Na `Main`, instanciar e chamar os métodos.

public interface Veiculo {
    void acelerar();
    void frear();
}

public class Carro implements Veiculo {
    @Override
    public void acelerar() {
        System.out.println("Carro acelerando...");
    }
    @Override
    public void frear() {
        System.out.println("Carro freando...");
    }
}

public class Main {
    public static void main(String[] args) {
        Carro meuCarro = new Carro();
        meuCarro.acelerar();
        meuCarro.frear();
    }
}

Aula 9: Acoplamento entre Classes

Esta aula final explora o conceito de acoplamento e coesão em POO, ensinando a projetar classes para um baixo acoplamento e alta coesão, resultando em um código mais flexível e fácil de manter.

Conteúdo Programático (15 min)

  • Acoplamento (Coupling): Grau de dependência entre classes.
  • Coesão (Cohesion): Foco de responsabilidade de uma classe.
  • Como reduzir acoplamento: Uso de interfaces.

Atividade Prática (25 min)

Refatorar um exemplo de alto acoplamento. Criar `Impressora` e `GerenciadorDocumentos` (acoplado à `Impressora`). Em seguida, introduzir uma interface `ServicoImpressao` para desacoplar o `GerenciadorDocumentos` da implementação concreta da impressora.

public interface ServicoImpressao {
    void imprimir(String texto);
}

public class Impressora implements ServicoImpressao {
    @Override
    public void imprimir(String texto) {
        System.out.println("Imprimindo: " + texto);
    }
}

public class GerenciadorDocumentos {
    private ServicoImpressao servicoImpressao; // Depende da interface

    public GerenciadorDocumentos(ServicoImpressao servicoImpressao) {
        this.servicoImpressao = servicoImpressao;
    }

    public void enviarParaImpressao(String documento) {
        servicoImpressao.imprimir(documento);
    }
}

public class Main {
    public static void main(String[] args) {
        Impressora hp = new Impressora();
        GerenciadorDocumentos gerenciador = new GerenciadorDocumentos(hp);
        gerenciador.enviarParaImpressao("Relatório Mensal");
    }
}
